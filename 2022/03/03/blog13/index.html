对象、方法(下)

封装继承多态

封装

属性私有，getset
作用：
1.提高程序安全性，保护数据
2.影藏代码的实现细节
3.统一接口
4.系统可维护性增加了

继承

在类后用extends指向父类
注意：
1.子类继承父类所有方法
2.私有的东西无法继承

  • super.name:
    • 指向父类的name
    • 只能出现在子类方法或者构造方法中
    • 只能在继承条件下使用
    • super()指父类的构造方法
  • this.name:
    • 指向当前类的name
    • 没继承也能使用
    • this()指当前类的构造方法
    • 相同点
    • 必须在代码第一行
    • super和this不能同时调用构造函数
方法重写

重写都是方法的重写，和属性无关
为什么要重写？：父类的功能不一定需要，不一定能满足

例：
B a = new A(); //子类重写了父类方法
a.test();

对于test方法：

  • 方法有static时，是类的方法，调用左边的B类。
  • 没有static时，是对象的方法，调用右边的A类。

所以：
1.只有非静态方法才是重写
2.方法名要相同
3.参数列表列表相同
3.修饰符范围可以扩大，不能缩小public>Protected>Default>private
4.抛出的异常范围可以缩小，不能扩大

多态

多态：父类的引用可以指向子类。即同一方法根据发送对象的不同采用多种不同的行为方式

一个对象的实际类型是确定的，可以指向的引用类型不确定
Student s1 = new Student();
Object s2 = new Student();

  • 父类只能调用自己和子类共有的方法，不能调用子类以独有的方法
  • 子类只能调用自己的和继承父类的方法
  • 多态是方法的多态
  • 父类和子类要有关系
  • 存在条件：方法有重写，继承关系，父类指向子类对象（static方法属于类，不能重写）
instanceof 判断一个对象是否有父子关系
类型转换

作用：方便方法的调用，可以减少重复的代码
1.高类型转换到低类型要强制转换(向下转型)

2.子类转换父类(向上转型)，会丢失方法
