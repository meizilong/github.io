封装继承多态

封装

属性私有，getset
作用：
1.提高程序安全性，保护数据
2.影藏代码的实现细节
3.统一接口
4.系统可维护性增加了

继承

在类后用extends指向父类
注意：
1.子类继承父类所有方法
2.私有的东西无法继承

  • super.name:
    • 指向父类的name
    • 只能出现在子类方法或者构造方法中
    • 只能在继承条件下使用
    • super()指父类的构造方法
  • this.name:
    • 指向当前类的name
    • 没继承也能使用
    • this()指当前类的构造方法
    • 相同点
    • 必须在代码第一行
    • super和this不能同时调用构造函数
方法重写

重写都是方法的重写，和属性无关
为什么要重写？：父类的功能不一定需要，不一定能满足

例：
B a = new A(); //子类重写了父类方法
a.test();

对于test方法：

  • 方法有static时，是类的方法，调用左边的B类。
  • 没有static时，是对象的方法，调用右边的A类。

所以：
1.只有非静态方法才是重写
2.方法名要相同
3.参数列表列表相同
3.修饰符范围可以扩大，不能缩小public>Protected>Default>private
4.抛出的异常范围可以缩小，不能扩大

多态

多态：父类的引用可以指向子类。即同一方法根据发送对象的不同采用多种不同的行为方式

一个对象的实际类型是确定的，可以指向的引用类型不确定
Student s1 = new Student();
Object s2 = new Student();

  • 父类只能调用自己和子类共有的方法，不能调用子类以独有的方法
  • 子类只能调用自己的和继承父类的方法
  • 多态是方法的多态
  • 父类和子类要有关系
  • 存在条件：方法有重写，继承关系，父类指向子类对象（static方法属于类，不能重写）
instanceof 判断一个对象是否有父子关系
类型转换

作用：方便方法的调用，可以减少重复的代码
1.高类型转换到低类型要强制转换(向下转型)

2.子类转换父类(向上转型)，会丢失方法

*值传递与引用传递的区别

方法重载

  • 1.方法名必须相同
  • 2.参数的参数类型、参数个数或参数顺序（同方法内的不同参数类型）必须不同
  • 3.返回类型、修饰符可以相同、也可以不同
  • 4.仅有返回值不同不足以区分两个方法体

    可变参数（不定向参数）

  • JDK 1.5开始，java支持传递同类型的可变参数
  • 在方法声明中，在指定参数类型后加一个省略号“…”
  • 一个方法中只能制定一个可变参数，他必须是方法的最后一个参数。任何普通的参数必须在它之前声明。

    递归

  • 递归头：什么时候不调用自己
  • 递归体：什么时候调用自身

运算符

位运算符

位运算用二进制计算，效率高。

  • a = 0011 1100
  • b = 0000 1101
  • a&b = 0000 1100
  • a|b = 0011 1101
  • a^b = 0011 0001
  • ~a = 1100 0011

2*8 计算机如何运算效率最高？

  • << 左移 相当于*2（二进制运算）
  • >> 右移 相当于/2

    2*2*2*2 左移四位即可

    扩展赋值运算符

  • a+=b //a=a+b
  • a-=b //a=a-b

    条件运算符

  • x ? y : z //如果x为真，结果为y，否则为z。(重要)

    ==注意：多用（）减少bug #F44336==
